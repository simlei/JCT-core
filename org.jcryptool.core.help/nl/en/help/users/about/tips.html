Tips and Tricks


Maximizing the window of a plugin

You might have opened the default view with editors, the context help, and some visualizations. If you then want one of them to be maximized just double-click on the tab (e.g. on the "Frequency Analysis" tab within the visualizations).


Using JCrypTool in a network environment or on a USB stick

In principle it is possible to use JCrypTool in a network environment or on an external storage medium like a USB stick or a USB hard disk. To enable this you have to do a minor change in one of the configuration files: It is necessary to change the path to the default workspace folder. This folder is used by all plug-ins to store your specific preferences.

Windows/ Linux:
Open the file JCrypTool.ini in your installation folder and search for the line -data @user.home/documents/.jcryptool. Replace @user.home/documents/.jcryptool with the absolute path to a folder available in your environment.

macOS:
The required changes are identical with the ones described above for Windows/Linux. However, the file JCrypTool.ini is located at JCrypTool.app\Contents\MacOS in your installation folder.


Updating JCrypTool

JCrypTool contains an integrated update mechanism. Use the Help menu for all update operations. This menu contains the entries Check for Updates and Install new Software. However, JCrypTool does not automatically search for new updates. You can either call the update search manually via the menu entry or you can change the Preferences on the Automatic Updates page to search automatically.


Updates are not visible

It is possible that even after a successful installation the updates/changes are not visible. This may occur for newly installed plug-ins as well as updates for existing ones. The reason for that is the JCrypTool workspace folder, which caches and stores all plug-in information. Sometimes this information is not updated and the data stored there does not reflect the current installation any more.

To solve this, you can either delete the .jcryptool folder in your document directory (inside your user home directory) manually or you can tell JCrypTool to do that job. For the first case (when trying to delete this folder), please remember that this folder is hidden on macOS and Linux. For the second case (to let JCrypTool delete this folder at each new start) simply add the parameter -clean at the end when launching JCrypTool (e.g. add it to the shortcut on Windows). This automatically deletes the workspace folder. This folder is automatically recreated and initialized when starting JCrypTool the next time. Remember to remove the parameter afterwards.

Here are two examples under Windows how the target command line call could lool like:
F:\Downloads\jcryptool\JCrypTool.exe -clean
F:\Downloads\jcryptool\JCrypTool.exe -nl de -data %LOCALAPPDATA%\jct-de -clean
